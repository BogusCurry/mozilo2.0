Es kann vorkommen das der adminTab "Plugins" oder auch das komplette Browserfenster leer ist obwohl Plugins enthalten sind. Dann ist ein nicht-kompatibles oder fehlerhaftes Plugin enthalten.

Lösungsansatz / Abhilfe: (Das Deaktivieren / Abschalten von Plugins ist hierfür nicht ausreichend.)

1. Alle Plugins aus dem Ordner "plugins" entfernen, also ausserhalb des Verzeichnisses (lokal) sichern.
2. Nacheinander die Plugins -einzeln- erneut installieren und testen bis der beschriebene Fehler wieder auftritt.
3. Dem Pluginautor mitteilen das sein Plugin fehlerhaft ist.


Icon Legende
{ICON_edit} Öffnet und schliesst die Inhaltsdetails eines Plugins.
{ICON_info} Per Mausklick öffnet sich die Beschreibung des Plugins.
{ICON_save} Wichtig: Speichern der aktuellen Plugin-Einstellungen.

In dieser Liste werden alle installierten Plugins angezeigt.

Nur wenn das Häkchen Aktivieren gesetzt ist, wird das Plugin arbeiten, anderenfalls kommt es zu keiner Ersetzung und der Pluginplatzhalter / Name wird angezeigt.

Achtung: Um die Änderungen eins Plugins zu speichern muss auf dieses Icon {ICON_save} geklickt werden. Es erfolgt keine Rückfrage.

Der Aufbau eines Plugins:

Ein Plugin besteht aus einem Ordner der den Namen des Plugins trägt. In dem Ordner muss mindestens eine "index.php"-Datei enthalten sein.
Es können je nach Funktion und Umfang aber auch wesentlich mehr Dateien und auch Unterordner sein.
{ICON_information} Hinweis: Ein Plugin ist kein PHP-include.


Beispiel 
Pluginname/                 Der Ordnername der so lautet wie der Plugin-Name.
           index.php        Ohne die obligatorische index.php-Datei ist
			    das Plugin nicht funktionsfähig.
